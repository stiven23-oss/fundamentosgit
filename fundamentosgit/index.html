Introducción

Git es un sistema de control de versiones distribuido que permite mantener un historial completo de cambios, trabajar en paralelo con ramas y colaborar de forma segura. Esta guía resume sus comandos principales, con ejemplos concisos.

“El buen diseño es obvio. El gran diseño es transparente.” – Joe Sparano

Esta página usa HTML5 (secciones, artículos, figuras, navegación) y CSS (grid de tarjetas, estilos tipográficos y componentes como .badge y .grid-cards).

Conceptos clave de Git

<Snapshots> y no diffs

Git guarda instantáneas (snapshots) del estado del proyecto en cada commit, lo cual hace las operaciones muy rápidas y confiables.

Cada commit apunta a su(s) padre(s), formando un grafo acíclico.

<Áreas> de trabajo

Git opera en tres áreas: Working Directory (archivos editándose), Staging Area (índice de lo que se va a confirmar) y Repository (historial persistente).

  • git add → mueve cambios al staging.
  • git commit → guarda un snapshot en el repositorio.
  • git restore/git reset → revierten o deshacen.

Comandos principales de Git

Básico git init

Crea el directorio oculto .git y empieza a rastrear versiones. 

$ git init
            Initialized empty Git repository in .git/

Usa git init -b main para fijar la rama inicial como main.

Básico git clone

Clona historial, ramas y configuración del remoto.

$ git clone https://servidor.com/usuario/proyecto.git
$ cd proyecto

Puedes clonar con --depth 1 para historial superficial.

Básico git add

$ git add archivo.html
$ git add .            # añade todo
$ git add -p           # por fragmentos (hunks)

Con -p puedes seleccionar partes específicas.

Básico git commit

$ git commit -m "feat: agrega sección de comandos básicos"
$ git commit --amend  # reescribe el último commit

Mensajes claros ayudan a entender el historial.

Sincronización fetch, pull, push

$ git fetch origin            # trae refs sin integrar
$ git pull --ff-only          # integra si posible fast-forward
$ git push origin main        # publica tu rama

pull = fetch + merge (por defecto) o rebase con --rebase.

Flujos de trabajo comunes

Feature branch (sencillo)

# crear rama y trabajar
git switch -c feature/login
# commits...
git push -u origin feature/login
# integrar
git switch main
git pull --ff-only
git merge feature/login
git push origin main

Recursos recomendados para Git

  • Ayuda integrada: git help <comando>, git <comando> -h
  • Convenciones de mensajes: estilo Conventional Commits para claridad (feat:, fix:, etc.).
  • Alias útiles (opcional): en ~/.gitconfig: 
    [alias]
  lg = log --oneline --graph --decorate --all
  co = checkout
  st = status -sb
  ci = commit
  br = branch
